# 同时制象棋

一款支持同时制对弈的在线象棋游戏，红黑双方同步决策、同时结算，体验紧张刺激的策略博弈。

## 游戏规则

### 核心玩法：同时制对弈

传统象棋是回合制，本游戏创新采用**同时制**：

1. **同步策略**：红黑双方**同时**思考和选择，互不知道对方的策略
2. **视角切换操作**：通过界面切换按钮分别操作红方/黑方棋子
3. **确认策略**：双方都准备好后，点击"确认策略"按钮
4. **同时结算**：双方移动同时执行，根据规则判定结果

> **注意**：同时制没有先手概念，双方地位完全平等。

### 胜负判定

| 结果 | 条件 |
|------|------|
| 红方胜 | 黑将被吃 |
| 黑方胜 | 红帅被吃 |
| 和棋 | 将帅同时被吃 |
| 同归于尽 | 双方移动到同一位置，双方棋子都被移除 |

### 吃子规则

#### 基础吃子
- 棋子移动到敌方棋子所在位置，即可吃子
- 己方棋子所在位置不能进入

#### 炮的吃子规则
- 炮必须隔**一个棋子**（炮架）才能吃子
- 移动路径上只能有炮架（任意一方），中间不能有其他棋子
- 移动到空位时，不能有子阻挡

### 同归于尽规则

当双方移动到**同一位置**时：
- 该位置上的双方棋子**同时被移除**
- 如果有一方是将帅，则对方获胜
- 如果双方都是将帅，则判为**和棋**

### 将帅对面规则

- 将和帅不能在同一列上直接面对面（中间无其他棋子阻挡）
- 如将帅同时移动导致面对面，双方同时被吃，判为**和棋**

### 防守反击规则（保护判定）

当一方吃子后，如果被吃的棋子的**同阵营其他棋子**能够反击到吃子棋子的新位置，则吃子方也被移除。

**示例**：
- 红炮吃掉黑卒 → 红炮移动到黑卒原位置
- 如果黑方有其他棋子（本回合未移动）能够吃掉红炮新位置 → 红炮被**防守反击**移除

> **注意**：只有本回合**未移动**的棋子才能进行防守反击。

### 长捉限制规则

为避免重复捉子，本游戏设置**长捉限制**：

- 连续3次用同一棋子捉同一目标 → 该行动被禁止
- "捉"：己方棋子能够吃到对方某个棋子
- 移动到安全位置或吃子后，长捉计数重置

### 碰撞处理

- 双方移动到同一位置 → 同归于尽
- 移动到对方棋子位置 → 正常吃子

## 棋子说明

| 棋子 | 红方 | 黑方 | 数量 | 行棋方式 |
|------|------|------|------|----------|
| 将/帅 | 帅 | 将 | 1 | 九宫内横竖一步 |
| 士/仕 | 仕 | 士 | 2 | 九宫内对角一步 |
| 象/相 | 相 | 象 | 2 | 走田字，不过河 |
| 马 | 马 | 马 | 2 | 走日字 |
| 车 | 车 | 车 | 2 | 横竖任意距离 |
| 炮 | 炮 | 炮 | 2 | 横竖任意距离，吃子需炮架 |
| 兵/卒 | 兵 | 卒 | 5 | 未过河前进一步，过河可横走 |

### 棋子详细行棋规则

#### 将/帅
- 只能在九宫内移动
- 每次移动一步，横竖方向
- 红帅在下方（row 7-9），黑将在上方（row 0-2）

#### 士/仕
- 只能在九宫内移动
- 每次移动一步，对角线方向
- 只能在九宫的5个特定位置活动

#### 象/相
- 走"田"字（对角线移动两格）
- 不能过楚河汉界（象不能飞过中线）
- 不能蹩象眼（田字中心有子则不能走）

#### 马
- 走"日"字（一格直线 + 一格对角）
- 不能蹩马腿（直线方向有子则不能走）

#### 车
- 横竖方向任意距离
- 不能有其他棋子阻挡

#### 炮
- 横竖方向任意距离
- 移动到空位：不能有子阻挡
- 吃子：必须隔一个棋子（炮架）才能吃子

#### 兵/卒
- 未过河：只能向前走一步
- 已过河（过楚河汉界）：可向前或横向走一步
- 红兵向下走（row 增加），黑卒向上走（row 减少）

## 快速开始

### 启动开发服务器

```bash
pnpm install
pnpm dev
```

启动后，在浏览器中打开 http://localhost:5000 查看应用。

### 构建生产版本

```bash
pnpm build
```

### Docker 部署

```bash
# 构建镜像
docker build -t xiangqi-simultaneous .

# 运行容器
docker run -d -p 5000:5000 xiangqi-simultaneous
```

## 项目结构

```
├── src/
│   ├── App.tsx           # 主应用组件
│   ├── ChessBoard.tsx    # 棋盘组件
│   ├── chessLogic.ts     # 行棋规则逻辑
│   ├── gameLogic.ts      # 游戏结算逻辑
│   ├── types.ts          # 类型定义
│   └── index.css         # 全局样式
├── server/               # 后端服务器
├── scripts/              # 构建脚本
└── Dockerfile            # Docker 配置
```

## 技术栈

- 前端：React 19 + TypeScript + Vite
- 后端：Express + TypeScript
- 通信：HTTP + WebSocket（联机模式）
- 样式：纯 CSS（无框架）
