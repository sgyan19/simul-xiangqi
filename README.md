# 同时制象棋

一款支持同时制对弈的在线象棋游戏，红黑双方各自暗自走棋，同时结算，体验紧张刺激的策略博弈。

## 游戏规则

### 核心玩法：同时制对弈

传统象棋是回合制，本游戏创新采用**同时制**：

1. **各自策略阶段**：红黑双方各自独立思考，选择棋子和目标位置
2. **视角切换操作**：通过界面切换按钮切换红方/黑方视角，分别操作双方棋子
3. **确认策略**：双方都确认后，点击"确认策略"按钮
4. **同时结算**：双方移动同时执行，判定胜负

### 胜负判定

| 结果 | 条件 |
|------|------|
| 红方胜 | 黑将/帅被吃 |
| 黑方胜 | 红帅/将被吃 |
| 和棋 | 将帅同时被吃（如面对面） |
| 同归于尽 | 双方移动到同一位置 |

### 行棋规则

本游戏遵循标准中国象棋行棋规则：

#### 将/帅
- 只能在九宫内移动
- 每次移动一步，横竖方向
- 红帅在下方（row 7-9），黑将在上方（row 0-2）

#### 士/仕
- 只能在九宫内移动
- 每次移动一步，对角线方向
- 只能在九宫的5个特定位置活动

#### 象/相
- 走"田"字（对角线移动两格）
- 不能过楚河汉界（象不能飞过中线）
- 不能蹩象眼（田字中心有子则不能走）

#### 马
- 走"日"字（一格直线 + 一格对角）
- 不能蹩马腿（直线方向有子则不能走）

#### 车
- 横竖方向任意距离
- 不能有其他棋子阻挡

#### 炮
- 横竖方向任意距离
- 移动到空位：不能有子阻挡
- 吃子：必须隔一个棋子（炮架）才能吃子

#### 兵/卒
- 未过河：只能向前走一步
- 已过河（过楚河汉界）：可向前或横向走一步
- 红兵向下走（row 增加），黑卒向上走（row 减少）

### 特殊规则

#### 将帅对面
- 将和帅不能在同一列上直接面对面（中间无子阻挡）
- 如将帅同时移动导致面对面，双方同时被吃，判为和棋

#### 碰撞处理
- 双方移动到同一位置：双方棋子同归于尽
- 移动到对方棋子位置：正常吃子

## 棋子说明

| 棋子 | 红方 | 黑方 | 数量 | 行棋方式 |
|------|------|------|------|----------|
| 将/帅 | 帅 | 将 | 1 | 九宫内横竖一步 |
| 士/仕 | 仕 | 士 | 2 | 九宫内对角一步 |
| 象/相 | 相 | 象 | 2 | 走田字，不过河 |
| 马 | 马 | 马 | 2 | 走日字 |
| 车 | 车 | 车 | 2 | 横竖任意距离 |
| 炮 | 炮 | 炮 | 2 | 横竖任意距离，吃子需炮架 |
| 兵/卒 | 兵 | 卒 | 5 | 未过河前进一步，过河可横走 |

## 快速开始

### 启动开发服务器

```bash
pnpm install
pnpm dev
```

启动后，在浏览器中打开 http://localhost:5000 查看应用。

### 构建生产版本

```bash
pnpm build
```

### Docker 部署

```bash
# 构建镜像
docker build -t xiangqi-simultaneous .

# 运行容器
docker run -d -p 5000:5000 xiangqi-simultaneous
```

## 项目结构

```
├── src/
│   ├── App.tsx           # 主应用组件
│   ├── ChessBoard.tsx    # 棋盘组件
│   ├── chessLogic.ts     # 行棋规则逻辑
│   ├── gameLogic.ts      # 游戏结算逻辑
│   ├── types.ts          # 类型定义
│   └── index.css         # 全局样式
├── server/               # 后端服务器
├── scripts/              # 构建脚本
└── Dockerfile            # Docker 配置
```

## 技术栈

- 前端：React 19 + TypeScript + Vite
- 后端：Express + TypeScript
- 通信：HTTP + WebSocket（联机模式）
- 样式：纯 CSS（无框架）
